This section is based on Justin Bois' fantastic data analysis course at Caltech.

We highly recommend you to read his introduction to bayesian parameter estimation to learn with more details what we will present here. Due to time constraints we will just go through a short primer on the topic.
As we mentioned during lecture for Bayesians probability is a measure of plausibility for any statement. For parameter estimation we want to find the probability distribution of the value of our parameters of interest, i.e. as Bayesians we will never say something like: "this is the value of the parameter we measure in our experiment with 100% certainty"; but rather we would report the probability distribution of the values that these parameters can take.

As researches we have to make decisions on what functional forms the likelihood and the prior will take. One of the things I like the most about Bayesian analysis is that every single assumption that goes into the analysis is transparent. Every researcher when performing statistical analysis is making assumptions about the data even when he/she doesn't now it. For example if you have ever used a student-t test you have implicitly assumed that your data is Gaussian distributed. But in the Bayesian framework we are under control of all of these assumptions, making the analysis much more clear.
Let's go ahead and list all of the assumptions that go into our analysis.

The likelihood

The likelihood, as the name indicates, tells us how likely it is that we would have observed our data given certain parameter value. In this case if my binding energy is completely off of what the most likely value is, then this term will be really small indicating that under our assumptions the data that we obtained would be very unlikely to be generated by our model given that parameter value. On the other hand if my binding energy is close to the most likely parameter value, then this term will be high, indicating that this specific choice of binding energy could have likely generated the data that we observed.
Let's list the assumptions that go into this term.

Every datum is independent of each other

The first safe assumption we will make is that each datum in our data set is completely independent of each other. Given the laws of probability we have the "mantra" that independence of events means to multiply probabilities. What that means mathematically is that we will write the likelihood as
where represents the datum and is read as the product of each datum in the data set D.

Gaussian likelihood for the deviations from our theoretical predictions

The likelihood contains all the information that we have on how the parameter we are trying to infer relates to our experiment. We could clasify likelihoods in two categories depending on the nature of the model we have for the data:
  1. The likelihood models the deviations from the model given some deterministic relationship between parameters and data.
  2. The likelihood itself could be the model if our relationship between parameters and data is probabilistic. For example if my model is itself a probability distribution of how I expect to find my data I could use that distribution directly as a model.
Our model falls into the first category. We have a deterministic relationship between binding energy and fold-change, i.e. our model doesn't tell us anything about the deviations from the mean fold-change. If we were interested in these deviations we would have to use a different theoretical framework such as chemical master equations (look at this interesting review by Jane Kondev for more on the use of chemical master equations). This means that our likelihood has to describe the distribution of deviations of the experimental data from our theoretical prediction. Note that for this we are assuming that the model indeed describes the data. Without getting into the philosophy of what does it mean for a model to be "correct" or not let's assume that the equation for fold-change is an accurate description of the phenomena.

Now that we have our likelihood ready let's move on to the other term, the prior.

The prior

The prior that now includes the extra parameter σ captures the information we knew about these parameters before performing the experiment (hence the name Prior Information). This is probably the single most controversial aspect about Bayesian statistics. Traditional statisticians claim that we can bias the outcome of our inference to be whatever we want given some arbitrary prior. But as a Bayesian, I think the prior is a natural extention of how we do science. It is important to acknowledge all the previous knowledge. There are other arguments one can give about the nature of the prior, for example:
  • Since we are writing one can think of it as an extra datum multiplying all of our data. That means given enough data if we choose a broad prior the likelihood will easily overcome the prior and the effect will be negligible.
  • There are information theoretic arguments on how to choose the most uninformative prior for each parameter. Therefore by using these uniformative priors we are guaranteed to commit the least possible to what we don't know about the parameter.

Finding the most likely parameter value

Let's say we want to find where the peak on that distribution is. That would give you what in the Bayesian jargon is called the MAP (for Maximum APosteriori estimation) or the most likely value of the parameter. This value is the special number that maximizes the posterior (and therefore the log posterior) distribution. Finding the extremum of a function is a common computational problem, and MATLAB has a lot of tools available to do this. This itself is an active research area on computer science, but for us we will use a simple function fminbnd to find the MAP value of our energy.
The fminbnd function takes 3 main arguments: the single parameter function to be optimized, and the lower and upper bound where to search for this minimum. In our case we want to maximize the log posterior, but since the function is designed to find minimums we can simply find the minimum of the negative log posterior. A tricky thing is that fminbnd takes a function with a single input argument and our logPosterior function takes 3 arguments as discussed before. The trick to work around this issue is to define what MATLAB calls an anonymous function. Here we will define a function that takes only the binding energy as input, having the data fixed in the definition of the function. This will make more sense once we code this.
%Define anonymous function to compute -logPosterior of the O2 data
minusLogPosteriorO2 = @(epsilonR) -logPosterior(dfO2.fold_change, dfO2.repressors, epsilonR);
% Test the function feeding a single parameter

Bayesian error bar for a parameter value.

One of the main differences between Bayesian and frequentist analysis is the interpretation of the parameter value and the "error bars" associated with these parameters. You might be familiar with the frase "95% confidence interval" that you might have read in a paper. Each school of thought has its own way to assign an error bar to the parameter value, but the interpretation of such constructs is completely different. If I were preachy about the Bayesian approach (which I'm trying really hard not to be) I would say that the Bayesian interpretation of probability and of this error bar assigned to an inferred parameter value makes more sense. As a matter of fact it is very likely that what you think the 95% confidence interval is telling you, which is a a frequentist construct, is actually what the Bayesian equivalent called the "credible region" is saying. To explain the difference between both interpretations I will adapt an example from this excellent video by Jake Vanderplas.
Imagine we are trying to infer the binding energy of our transcription factor. Under the frequentist interpretation the transcription factor has a specific value for the binding energy. As we take experimental measurements we can report what frequentists call the confidence interval. This confidence interval has a very non-intuitive interpretation. What the 95% confidence interval is telling you is a probabilistic statement about the confidence interval itself. In other words, the confidence interval tells you a recipe in which in the limit of many many many experimental repeats, for every confidence interval you construct for such an experiment, 95% of the time this interval would contain the "real" value of your parameter. It is weird, right? If you are confused right now on what I tried to say it means that I'm doing a great job in transmitting the message that the confidence interval is not what most people think it is.
Let me try to rephrase in a different way. Frequentist statistics gives you a "recipe" to construct an interval called the confidence interval. The interpretation of this confidence interval is associated with the recipe itself rather than with the actual interval you obtain when following the recipe. What the recipe tells you is that in the limit of infinitely many experimental repeats, 95% of the times you follow the recipe to construct the confidence interval, this interval would actually contain the value for your parameter. I know... it is shocking and you feel like you've been living in the darkness all of your life. I call that effect the "Bayesian baptism" (but I'm not being preachy!).
In the Bayesian interpretation one never says "the value of the parameter is exactly x." Since we assign a probability distribution to the parameter value, we would say that the value of the parameter has a probabilityx of falling in certain interval. In other words if we report what Bayesians call the 95% credible region what we are reporting is that given our data, the parameter we are trying to infer has a 95% probability of falling within the credible region range. That is a much more natural assignment of an error bar on an inferred parameter, I think. It shouldn't be surprising as I mentioned before that what you thought the confidence interval was telling you is actually what the credible region is saying. Giving Bayesian interpretations to frequentist constructs is a very common mistake.

Computing the error bar in the binding energy

To numerically compute the second derivative of a function there are several routines one could follow. The field of numerical differentiation is it's own beast. If you are interested in how to generate a better estimate of this derivative there is plenty of literature on the topic. For simplicity we will take the most naive and to some extent innacuarte approach. What we will do is to use the function del2 to compute the discrete Laplacian, i.e. the second derivative. Therefore we will define a range of binding energies around the MAP, compute this discrete Laplacian, and take the value of the one closest to the MAP.
% Define very fine range of binding energies
energies = linspace(O2energy - 0.3, O2energy + 0.3, 200);
% Initialize array to save the log posterior probability
logP = zeros(1, length(energies));
% Loop through the energies computing the log probability at each step.
for i=1:length(logP)
logP(i) = logPosterior(dfO2.fold_change, dfO2.repressors, energies(i));
end %for
% Compute the discrete Laplacian
% The second argument of the function is the spacing between gridpoints.
% This is necessary when computing the numerical derivative
laplacian = del2(logP, min(diff(energies)));
% Find the closest value in energies to the O2energy
[closest, idx] = min(abs(energies - O2energy));
% Compute the variance for the binding energy parameter distribution
sigmaSquareEnergy = - 1 / laplacian(idx);
% Compute the error bar, i.e. the standard deviation of the distribution
errorEnergy = sqrt(sigmaSquareEnergy)
errorEnergy = 0.0814
